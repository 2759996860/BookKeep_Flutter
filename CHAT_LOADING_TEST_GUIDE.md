## 🔍 常见问题排查

### 问题1：上拉不触发加载

**检查点**：
1. 控制台是否有 `📢 触发上拉加载更多` 日志？
   - **有** → 说明触发成功，检查网络请求
   - **无** → 说明滚动监听未触发

2. 如果没有触发，检查：
   - `_hasMoreMessages` 是否为 `true`？
   - `_isLoadingMore` 是否为 `false`？
   - `_messages.isNotEmpty` 是否为 `true`？

**解决方案**：
```dart
// 在调试控制台中查看状态
print('_hasMoreMessages: $_hasMoreMessages');
print('_isLoadingMore: $_isLoadingMore');
print('_messages.length: ${_messages.length}');
```

---

### 问题2：加载图标不显示

**检查点**：
1. `_isLoadingMore` 是否在加载期间为 `true`？
2. `itemCount` 是否正确增加了1？

**调试方法**：
在 `itemBuilder` 中添加日志：
```dart
itemBuilder: (context, index) {
  print('渲染索引: $index, 消息总数: ${_messages.length}, 加载中: $_isLoadingMore');
  
  if (index == _messages.length) {
    print('✅ 渲染加载提示项');
    return LoadingIndicator();
  }
  // ...
}
```

---

### 问题3：Token刷新仍然失败

**检查点**：
1. 控制台是否有 `🔄 ChatService: 检测到401错误`？
2. 是否有 `✅ Token刷新并保存成功`？
3. 重试后的状态码是多少？

**可能原因**：
- RefreshToken也过期了 → 需要重新登录
- 网络连接失败 → 检查网络
- 服务器返回500错误 → 检查后端日志

## 🐛 已知限制

1. **弹性滚动依赖**：上拉加载依赖于Flutter的弹性滚动效果（BouncingScrollPhysics），在某些Android设备上可能需要更明显的上拉动作。

2. **网络延迟**：如果网络较慢，加载提示会显示较长时间，建议添加超时处理（当前为10秒）。

3. **内存占用**：如果消息非常多（如1000+条），全部加载到内存可能占用较多资源。建议考虑虚拟列表优化（未来优化方向）。
